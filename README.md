## 项目简介

这是一个**纯前端、零后端、零数据库**的场内ETF动态定投监控系统。基于以下理念设计：

- 📊 **实时监控**：4只场内ETF的实时价格、估值分位、股息率
- 🧮 **智能分配**：每月 ¥8,000 定投资金根据估值档位动态分配
- ⚖️ **自动再平衡**：偏离目标仓位 ±15% 自动生成调仓清单
- 📱 **全平台适配**：手机/平板/电脑自动响应式布局
- 🌓 **双主题**：深色金融模式（默认）/ 浅色模式一键切换

**技术栈**：HTML5 + Tailwind CSS v3 + Vanilla JavaScript + Chart.js，纯静态页面，GitHub Pages 直接运行。

## 核心策略

### 长期目标基准仓位

| 资产 | 代码 | 目标占比 | 角色定位 |
|------|------|---------|---------|
| 0-3年国债政金债ETF | 511580 | **10%** | 压舱石·流动性缓冲 |
| 中证红利低波动ETF | 512890 | **40%** | 核心收益引擎 |
| 国泰纳斯达克100ETF | 513100 | **35%** | 成长弹性来源 |
| 华安黄金ETF | 518880 | **15%** | 尾部风险对冲 |

### 为什么不是 25/25/25/25？

均等持仓忽视资产间风险差异。本组合借鉴桥水全天候策略框架，根据各类资产的风险收益特征设计差异化权重：

- **短债 10%**：提供确定性票息，是组合"安全气囊"，非主要收益来源
- **红利 40%**：A股高股息策略长期年化 8-10%，波动可控，担纲组合最大权重
- **纳指 35%**：全球顶级科技资产，长期年化 12-15%，高波动换高回报
- **黄金 15%**：股债双杀时逆势上涨，对冲尾部风险

### 10%短债够用吗？（三重缓冲机制）

| 层级 | 缓冲来源 | 说明 |
|------|---------|------|
| 第一层 | 每月新增 ¥8,000 | 年化 ¥96,000 持续现金流 ≈ 组合 10% 的新弹药 |
| 第二层 | 15%黄金对冲 | 美股大跌时金价往往上涨，可止盈加仓权益 |
| 第三层 | 年度再平衡 | 牛市纳指超配 → 年末卖出 → 自动回流短债 |

## GitHub Pages 部署步骤

### 第一步：创建 GitHub 仓库

```bash
# 1. 在 GitHub 新建仓库，名称如：etf-dca-monitor
# 2. 仓库设置为 Public（公开）
```

### 第二步：上传代码

```bash
# 克隆仓库到本地
git clone https://github.com/YOUR_USERNAME/etf-dca-monitor.git
cd etf-dca-monitor

# 将所有文件复制到仓库目录
# 包括：index.html, config.js, README.md 等

# 提交并推送
git add .
git commit -m "Initial commit: 四资产ETF动态定投监控系统"
git push origin main
```

### 第三步：启用 GitHub Pages

1. 进入仓库 → **Settings** → **Pages**
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main`，目录选择 `/ (root)`
4. 点击 **Save**
5. 等待1-2分钟，页面 URL 将显示在 Settings > Pages 顶部

### 第四步：配置 GitHub Actions 自动部署（可选）

仓库已包含 `.github/workflows/deploy.yml`，推送代码后自动部署到 Pages。

## 使用教程

### 日常操作流程

1. **首次使用**：在"持仓控制面板"输入四类ETF的持有股数，点击"保存持仓"
2. **每月20日晚**：打开页面查看"月度资金分配"模块，确认下月各标的买入金额
3. **每月21日开盘**：在海通e海通财APP执行定投
4. **盘中查看**：页面数据每 5/10/30 分钟自动刷新（可在顶部下拉选择间隔）

### 读懂页面信号

| 信号 | 含义 | 操作 |
|------|------|------|
| 🟢 绿色标签 | 低估加仓 | 提高该标的月度定投额 |
| 🟡 黄色标签 | 中性正常 | 按基准金额定投 |
| 🔴 红色标签 | 高估预警 | 暂停/减半该标的定投 |
| 🔵 蓝色标签 | 固收资产 | 短债固定规则 |
| ⚠️ 再平衡提醒 | 12月调仓期 | 核对仓位，偏离超±15%需调仓 |

## 配置说明

所有投资策略参数集中在 `config.js` 文件中，用户可自由修改：

```javascript
// config.js 主要可配置项：

CONFIG.monthlyTotal = 8000;           // 每月定投总额（元）
CONFIG.targetAllocation;              // 目标仓位百分比
CONFIG.baseAllocation;                // 基准月分配金额
CONFIG.nasdaq.peOvervalued = 34;      // 纳指高估PE阈值
CONFIG.nasdaq.peUndervalued = 25;     // 纳指低估PE阈值
CONFIG.dividend.pbPercentileOvervalued = 80; // 红利高估PB分位
CONFIG.gold.bubbleGain = 0.20;        // 黄金泡沫涨幅阈值
CONFIG.rebalance.deviationThreshold = 0.15;  // 再平衡偏离阈值
CONFIG.refresh.defaultInterval = 300000;     // 默认刷新间隔(ms)
```

> ⚠️ 修改配置后刷新页面即生效，无需改动 index.html 业务逻辑代码。

## 海通e海通财定投实操

### 设置定时条件单

1. 打开 **e海通财 APP** → **交易** → **条件单**
2. 新建条件单 → 选择 **"定时买入"**
3. 选择标的（如 512890 红利低波ETF）
4. 设置触发时间：**每月21日 09:35**（开盘后5分钟）
5. 委托价格：即时卖一价
6. 委托数量：参考系统分配金额 ÷ 实时价格（向下取整至100股）
7. 有效期：长期有效

### 调整定投金额

根据系统"月度资金分配"模块的提示：

- 纳指高估 → 暂停 513100 条件单，调高 512890/518880 买入金额
- 深度低估 → 暂停 511580 条件单，集中买入低估标的
- 黄金泡沫 → 暂停 518880 条件单
