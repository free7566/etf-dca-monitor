逐一检查四类资产的估值标签：
- 🔴 红色标签 → 重点预警
- 🟡 黄色标签 → 正常运行
- 🟢 绿色标签 → 加仓机会
- 🔵 蓝色标签 → 固收正常

## 第三步：资金分配核对

- 记录任何异常偏差

1. 查看"持仓仓位 & 再平衡测算"模块
2. 检查是否有偏离超±15%的资产
3. 如是12月，额外确认年末再平衡提醒是否触发

## 第五步：预警汇总
收集所有预警信号，按严重程度排序：
- 🔴 P0 紧急：极端泡沫、减仓信号
- 🟠 P1 重要：高估、暂停定投
- 🟡 P2 关注：对冲行情、再平衡提醒
- 🟢 P3 利好：低估加仓机会

## ⏰ 定时任务配置

### 每日定时巡检

```yaml
# Wokao Bot 定时任务配置
name: ETF监控每日巡检
schedule: "0 9,12,15,20 * * 1-5"  # 工作日 9:00/12:00/15:00/20:00
timezone: Asia/Shanghai
task:
  type: web_monitor
  url: https://YOUR_USERNAME.github.io/etf-dca-monitor/
  prompt: 执行四资产ETF每日巡检，按标准格式输出报告
  notify:
    - method: push  # 推送通知
      condition: on_alert  # 仅在触发预警时通知
```

### 定投日专项检查（每月21日）

```yaml
name: 定投日专项检查
schedule: "0 8 21 * *"  # 每月21日 08:00
timezone: Asia/Shanghai
task:
  type: web_monitor
  url: https://YOUR_USERNAME.github.io/etf-dca-monitor/
  prompt: |
    今天是定投日（每月21日），请执行专项检查：
    1. 确认月度资金分配方案正确
    2. 生成每只ETF的具体买入股数：
       买入股数 = 分配金额 ÷ 最新净值（向下取整到100股）
    3. 列出海通e海通财条件单操作清单
  notify:
    - method: push
```

### 收盘后检查（工作日15:30）

```yaml
name: 收盘估值复核
schedule: "0 15 30 * * 1-5"  # 工作日 15:30
timezone: Asia/Shanghai
task:
  type: web_monitor
  url: https://YOUR_USERNAME.github.io/etf-dca-monitor/
  prompt: |
    收盘后检查：
    1. 当日各ETF涨跌幅汇总
    2. 估值档位是否有变化
    3. 是否需要调整明日定投计划
    4. 检查纳指回撤是否触发加仓
```
